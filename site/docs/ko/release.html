릴리즈 가이드

@siisee11/discode와 플랫폼 바이너리 패키지를 배포하는 유지보수자 흐름입니다. docs/RELEASE_NPM.ko.md를 기반으로 정리했습니다.

1) 사전 점검

  • npm 계정이 siisee11 조직에 publish 권한이 있는지 확인합니다.
  • 2FA 사용 시 npm automation token 사용을 권장합니다.
  • npm 토큰은 절대 노출하거나 커밋하지 마세요.
npm whoami
npm token list

2) 버전 올리기

아래 값을 같은 버전으로 함께 올립니다:

  • package.json의 version
  • optionalDependencies의 @siisee11/discode-* 버전들

3) 빌드 및 패키징

npm run typecheck
npm run build
npm run build:release
npm run pack:release

산출물은 dist/release/에 생성됩니다.

4) 패키지 배포

플랫폼 패키지를 먼저 배포한 뒤 메타 패키지를 배포합니다.

npm publish --access public --workspaces=false dist/release/discode-darwin-arm64
npm publish --access public --workspaces=false dist/release/discode-darwin-x64
npm publish --access public --workspaces=false dist/release/discode-linux-x64
npm publish --access public --workspaces=false dist/release/discode-windows-x64

npm publish --access public --workspaces=false dist/release/npm/discode

위 명령은 예시입니다. 실제 릴리즈에서는 생성된 모든 플랫폼 패키지를 빠짐없이 배포하세요.

5) 배포 검증

npm view @siisee11/discode version
npm i -g @siisee11/discode@latest
discode --version

자주 발생하는 배포 이슈

  • EOTP: automation token이 아니거나 2FA 정책 영향일 수 있습니다.
  • You cannot publish over the previously published versions: 버전이 이미 존재합니다.
  • Access token expired or revoked: npm 토큰을 재설정 후 다시 시도하세요.
